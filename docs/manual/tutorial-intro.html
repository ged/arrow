Arrow: Applications Tutorial: Introduction

Architecture

Arrow is an application server that is designed somewhat like a J2EE servlet container, but using Ruby as the language of implementation instead of Java. There are some differences, of course, but developers familiar with the J2EE servlet specification will notice some similarities.

Overview

The Arrow architecture is made up of four basic parts: the Dispatcher, the Broker, one or more Applications, and Transactions which are wrapped around incoming requests.

Arrow Classes

The Arrow appserver framework is made up of the following base classes:

Arrow::Dispatcher
This class is the actual mod_ruby request handler; it loads the configuration, sets up the Broker, and reloads the configuration if it has changed since the last time it was read, as well as creating Transaction objects for incoming requests and rendering the resulting output.
Arrow::Transaction
Encapsulates the incoming request, providing easy access to request values, configuration values, session data, and validated query arguments.
Arrow::Broker
Decides how Transaction objects get handled by loading and building a registry of Application objects.

Arrow: Installation

As with most things, the hardest part of getting Arrow onto your machine is the dependencies. Depending on your operating system, many of these will have packages for easier installation. It's worth checking with all of these, but be careful to verify the versions are greater than or equal to what we list as required. Otherwise, we provide here links to what we believe to be the most authoritative source of each.

Apache

All development and testing is done using the Apache 1.3.x series webserver on Linux, FreeBSD, and MacOS X, though you should be able to run it under other platforms and/or Apache 2.0.x. We'd love to hear about you experiences under other configurations, and patches for portability are of course always welcomed.

The Generic Apache Request Library (libapreq)

Arrow requires the libapreq extensions to mod_ruby, which in turn requires that you install the Generic Apache Request Library, or libapreq.

Ruby

Using Ruby (1.8.1) is strongly urged, as the majority of our work has been done under the development version of ruby.

Arrow has also been tested and seems to work under the 1.8.0 release, but that version of Ruby has some known bugs which are fixed in later versions, so if you're having trouble, try a more recent Ruby.

mod_ruby

The mod_ruby Apache module embeds a Ruby interpreter into Apache, much like mod_perl and mod_python do for their respective interpreters.

Arrow depends on the libapreq extensions to mod_ruby, support for which was added in version 1.2.0. The current version as of this writing is mod_ruby-1.2.2. Arrow will probably work with earlier 1.2.x releases, too, but please upgrade to the most current one before reporting any problems you might encounter.

Ruby Library Dependencies

These are the various ruby libraries you will need to install in addition to the standard distributed ones. Known good versions of each one are in the distribution in the redist/ directory, but you may wish to install newer or customized versions.

  • Ruby-Cache (0.3)
  • FormValidator (0.1.3)
  • HashSlice (1.03)
  • PluginFactory (0.01)

Here also are some standard libraries that are used but may not be included with your ruby distribution (debian, for instance).

  • strscan (0.6.7)

Optionally, you can include the following libraries (with a word on what added functionality they bring in).

  • Test-Unit-Mock (0.03) - to run the tests.
  • Diff (0.4) - used by TestUnitMock.
  • posixlock - Required if you wish to use the 'posix' type of session locking.

Installing the Arrow Libraries

Now that all of the dependencies are installed (and you have obtained the Arrow sources), you can optionally test Arrow by running the included unit tests (which should all pass...):

$ ./test.rb

0 patterns given on the command line
Required 11 files.
Loaded suite Arrow Web Application Test Suite
Started
........................................................................
.....................
Finished in 1.167613 seconds.

103 tests, 2276 assertions, 0 failures, 0 errors

Finally, put the Arrow ruby libraries in place with the provided installer (you may need root privileges to do this):

$ ./install.rb

Arrow Installer Revision: 1.2 
Testing for the Ruby-Cache library...found.
Testing for the FormValidator library...found.
Testing for the HashSlice library...found.
Testing for the Strscan library...found.
Installing...done.

This is going to put the Arrow libraries in place, but will not have set up any default applets, templates or configuration files. These will need to be added as we configure things.
